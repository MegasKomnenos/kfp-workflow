# Katib Manifests

This directory is the package-local landing area for manifests generated by `mambasl-new katib render`.

Render a reviewable manifest:

```bash
mambasl-new katib render \
  --spec configs/experiments/fd_all_core_default.yaml \
  --dataset FD001 \
  --output kubeflow/katib/fd001_default_katib.yaml
```

Submit directly from the package CLI:

```bash
mambasl-new katib submit \
  --spec configs/experiments/fd_all_core_default.yaml \
  --dataset FD001
```

Preview the generated manifest without applying it:

```bash
mambasl-new katib submit \
  --spec configs/experiments/fd_all_core_default.yaml \
  --dataset FD001 \
  --dry-run
```

This is package-local Katib flow only. If you are operating the integrated root stack, use the root [README.md](/home/scouter/proj_2026_1_etri/test/README.md) and [OPERATIONS.md](/home/scouter/proj_2026_1_etri/test/OPERATIONS.md) instead.
