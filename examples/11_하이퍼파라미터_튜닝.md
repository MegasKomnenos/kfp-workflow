# 11. 하이퍼파라미터 튜닝

## 현재 권장 흐름

루트 사용자 관점의 기본 튜닝 진입점은 다음입니다.

```bash
kfp-workflow tune --spec configs/tuning/mambasl_cmapss_tune.yaml
```

이 명령은 Katib 실험을 생성해 제출합니다.

## 탐색 공간 확인

```bash
kfp-workflow tune space --spec configs/tuning/mambasl_cmapss_tune.yaml
```

프로필 변경:

```bash
kfp-workflow tune space \
  --spec configs/tuning/mambasl_cmapss_tune.yaml \
  --set hpo.builtin_profile=aggressive
```

## 드라이런과 대기

```bash
kfp-workflow tune \
  --spec configs/tuning/mambasl_cmapss_tune.yaml \
  --dry-run

kfp-workflow tune \
  --spec configs/tuning/mambasl_cmapss_tune.yaml \
  --wait
```

## 상태와 결과

```bash
kfp-workflow tune status
kfp-workflow tune status <experiment-id-or-prefix>
kfp-workflow tune logs <experiment-id-or-prefix>
kfp-workflow tune results <experiment-id-or-prefix>
```

최적 파라미터를 기존 pipeline spec에 반영:

```bash
kfp-workflow tune results <experiment-id-or-prefix> \
  --apply-best configs/pipelines/mambasl_cmapss_smoke.yaml
```

## 부트스트랩

결과 PVC가 필요하면 먼저 준비합니다.

```bash
kfp-workflow cluster bootstrap \
  --type tune \
  --spec configs/tuning/mambasl_cmapss_tune.yaml
```

## 주의

- 문서 기준의 공개 흐름은 `tune --spec`, `tune space`, `tune status`, `tune results`, `tune logs` 입니다.
- `tune run`, `tune katib`, `tune show-space`는 코드에 남아 있는 숨겨진 호환 경로이므로 새 문서나 자동화의 기본 인터페이스로 삼지 않는 편이 안전합니다.
