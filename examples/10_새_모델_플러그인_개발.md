# 10. 새 모델 플러그인 개발

## 개요

kfp-workflow의 플러그인 시스템은 새로운 모델을 파이프라인에 통합하는 표준화된 방법을 제공합니다. `ModelPlugin` 추상 기반 클래스(ABC)를 구현하기만 하면, 기존의 파이프라인 컴파일러, CLI, 서빙 인프라를 그대로 활용할 수 있습니다.

### 플러그인 아키텍처 철학

- **관심사 분리**: ML 로직은 플러그인에, 인프라 로직은 프레임워크에
- **계약 기반**: 추상 인터페이스가 입출력 계약을 강제
- **데이터 직렬화**: 스테이지 간 데이터는 PVC의 파일로 전달, 메타데이터만 JSON으로 이동
- **독립 패키징**: 모델 코드는 별도 Python 패키지로 유지 가능

## ModelPlugin ABC

모든 플러그인은 `ModelPlugin` ABC를 구현해야 합니다. 5개의 파이프라인 스테이지 + 추론 메서드로 구성됩니다:

```
┌──────────┐    ┌────────────┐    ┌─────────┐    ┌──────────┐    ┌────────────┐
│ load_data│───>│ preprocess │───>│  train  │───>│ evaluate │───>│ save_model │
└──────────┘    └────────────┘    └─────────┘    └──────────┘    └────────────┘
LoadDataResult  PreprocessResult  TrainResult    EvalResult      SaveResult
```

### 필수 구현 메서드

| 메서드 | 입력 | 출력 | 역할 |
|--------|------|------|------|
| `name()` | — | `str` | 플러그인 고유 식별자 |
| `load_data()` | spec, data_mount_path | `LoadDataResult` | 데이터 로드/다운로드 |
| `preprocess()` | spec, LoadDataResult, artifacts_dir | `PreprocessResult` | 전처리, .npy 저장 |
| `train()` | spec, PreprocessResult, model_dir | `TrainResult` | 모델 학습, 가중치 저장 |
| `evaluate()` | spec, TrainResult, PreprocessResult | `EvalResult` | 테스트셋 평가 |
| `save_model()` | spec, TrainResult, EvalResult, final_model_dir | `SaveResult` | 최종 모델 저장/등록 |
| `predict()` | model_path, input_data, model_config | Any | 추론 (KServe용) |

### 선택적 구현 메서드 (Config 스키마)

| 메서드 | 반환 | 역할 |
|--------|------|------|
| `model_config_schema()` | Pydantic Model 또는 None | model.config 검증 스키마 |
| `dataset_config_schema()` | Pydantic Model 또는 None | dataset.config 검증 스키마 |
| `train_config_schema()` | Pydantic Model 또는 None | train 섹션 검증 스키마 |

## 단계별 구현 가이드

### 1단계: 모델 패키지 생성

새 모델을 `models/` 디렉토리에 별도 Python 패키지로 생성합니다:

```
models/
└── my-new-model/
    ├── pyproject.toml
    ├── README.md
    └── src/
        └── my_new_model/
            ├── __init__.py
            ├── model.py       # 모델 아키텍처
            ├── data.py        # 데이터 로딩
            ├── preprocess.py  # 전처리
            └── train.py       # 학습 루프
```

`pyproject.toml` 예시:

```toml
[build-system]
requires = ["setuptools>=68.0", "wheel"]
build-backend = "setuptools.backends._legacy:_Backend"

[project]
name = "my-new-model"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = [
    "torch>=2.0",
    "numpy>=1.24",
]
```

### 2단계: 플러그인 어댑터 작성

`src/kfp_workflow/plugins/` 디렉토리에 어댑터 파일을 생성합니다:

```python
# src/kfp_workflow/plugins/my_new_model.py
"""My New Model plugin adapter."""

from __future__ import annotations

import json
import shutil
from pathlib import Path
from typing import Any, Dict

from pydantic import BaseModel, ConfigDict

from kfp_workflow.plugins.base import (
    EvalResult,
    LoadDataResult,
    ModelPlugin,
    PreprocessResult,
    SaveResult,
    TrainResult,
)


# ---------------------------------------------------------------------------
# Config 스키마 (선택 사항이지만 권장)
# ---------------------------------------------------------------------------

class MyModelConfig(BaseModel):
    """model.config 검증 스키마."""
    hidden_dim: int = 128
    num_layers: int = 4
    dropout: float = 0.1

    model_config = ConfigDict(extra="allow")


class MyDatasetConfig(BaseModel):
    """dataset.config 검증 스키마."""
    dataset_name: str = "default"
    split_ratio: float = 0.8

    model_config = ConfigDict(extra="allow")


# ---------------------------------------------------------------------------
# 플러그인 구현
# ---------------------------------------------------------------------------

class MyNewModelPlugin(ModelPlugin):
    """My New Model 플러그인."""

    @staticmethod
    def name() -> str:
        return "my-new-model"

    @classmethod
    def model_config_schema(cls):
        return MyModelConfig

    @classmethod
    def dataset_config_schema(cls):
        return MyDatasetConfig

    # -- Stage 1: load_data ------------------------------------------------

    def load_data(
        self,
        spec: Dict[str, Any],
        data_mount_path: str,
    ) -> LoadDataResult:
        from my_new_model.data import download_and_load

        ds_cfg = spec["dataset"].get("config", {})
        data_dir = str(Path(data_mount_path) / ds_cfg.get("dataset_name", "default"))

        train_count, test_count = download_and_load(data_dir)

        return LoadDataResult(
            data_dir=data_dir,
            dataset_name=ds_cfg.get("dataset_name", "default"),
            num_train_samples=train_count,
            num_test_samples=test_count,
        )

    # -- Stage 2: preprocess -----------------------------------------------

    def preprocess(
        self,
        spec: Dict[str, Any],
        load_result: LoadDataResult,
        artifacts_dir: str,
    ) -> PreprocessResult:
        import numpy as np
        from my_new_model.preprocess import prepare_arrays

        out = Path(artifacts_dir)
        out.mkdir(parents=True, exist_ok=True)

        arrays = prepare_arrays(
            data_dir=load_result.data_dir,
            config=spec.get("dataset", {}).get("config", {}),
        )

        paths = {}
        for name, arr in arrays.items():
            p = out / f"{name}.npy"
            np.save(str(p), arr)
            paths[name] = str(p)

        return PreprocessResult(
            artifacts_dir=str(out),
            x_train_path=paths["x_train"],
            y_train_path=paths["y_train"],
            x_val_path=paths["x_val"],
            y_val_path=paths["y_val"],
            x_test_path=paths["x_test"],
            y_test_path=paths["y_test"],
            feature_dim=int(arrays["x_train"].shape[-1]),
            seq_len=int(arrays["x_train"].shape[1]),
            num_train=int(arrays["x_train"].shape[0]),
            num_val=int(arrays["x_val"].shape[0]),
            num_test=int(arrays["x_test"].shape[0]),
        )

    # -- Stage 3: train ----------------------------------------------------

    def train(
        self,
        spec: Dict[str, Any],
        preprocess_result: PreprocessResult,
        model_dir: str,
    ) -> TrainResult:
        import numpy as np
        import torch
        from my_new_model.train import train_loop

        model_cfg = spec["model"].get("config", {})
        train_cfg = spec.get("train", {})

        x_train = np.load(preprocess_result.x_train_path)
        y_train = np.load(preprocess_result.y_train_path)
        x_val = np.load(preprocess_result.x_val_path)
        y_val = np.load(preprocess_result.y_val_path)

        model, best_epoch, train_loss, val_loss = train_loop(
            x_train, y_train, x_val, y_val,
            model_cfg=model_cfg,
            max_epochs=train_cfg.get("max_epochs", 50),
            lr=train_cfg.get("learning_rate", 1e-3),
        )

        out = Path(model_dir)
        out.mkdir(parents=True, exist_ok=True)
        model_path = str(out / "model.pt")
        torch.save(model.state_dict(), model_path)

        return TrainResult(
            model_path=model_path,
            best_epoch=best_epoch,
            train_loss=train_loss,
            val_loss=val_loss,
        )

    # -- Stage 4: evaluate -------------------------------------------------

    def evaluate(
        self,
        spec: Dict[str, Any],
        train_result: TrainResult,
        preprocess_result: PreprocessResult,
    ) -> EvalResult:
        import numpy as np
        from my_new_model.train import evaluate_model

        x_test = np.load(preprocess_result.x_test_path)
        y_test = np.load(preprocess_result.y_test_path)

        metrics = evaluate_model(
            model_path=train_result.model_path,
            x_test=x_test,
            y_test=y_test,
            model_cfg=spec["model"].get("config", {}),
        )

        return EvalResult(
            metrics=metrics,
            model_path=train_result.model_path,
        )

    # -- Stage 5: save_model -----------------------------------------------

    def save_model(
        self,
        spec: Dict[str, Any],
        train_result: TrainResult,
        eval_result: EvalResult,
        final_model_dir: str,
    ) -> SaveResult:
        model_name = spec["model"]["name"]
        model_version = spec["model"].get("version", "v1")

        final = Path(final_model_dir)
        final.mkdir(parents=True, exist_ok=True)

        dst = final / "model.pt"
        shutil.copy2(train_result.model_path, str(dst))

        return SaveResult(
            saved_path=str(dst),
            model_name=model_name,
            model_version=model_version,
        )

    # -- Inference ----------------------------------------------------------

    def predict(
        self,
        model_path: str,
        input_data: Any,
        model_config: Dict[str, Any],
    ) -> Any:
        import torch
        from my_new_model.model import MyModel

        model = MyModel(**model_config)
        state = torch.load(model_path, map_location="cpu", weights_only=True)
        model.load_state_dict(state)
        model.eval()

        with torch.no_grad():
            tensor = torch.tensor(input_data, dtype=torch.float32)
            return model(tensor).numpy()
```

### 3단계: 레지스트리에 등록

`src/kfp_workflow/plugins/__init__.py`의 `_build_registry()` 함수에 새 플러그인을 추가합니다:

```python
def _build_registry() -> Dict[str, Type[ModelPlugin]]:
    registry: Dict[str, Type[ModelPlugin]] = {}

    # --- MambaSL C-MAPSS ---
    from kfp_workflow.plugins.mambasl_cmapss import MambaSLCmapssPlugin
    registry[MambaSLCmapssPlugin.name()] = MambaSLCmapssPlugin

    # --- My New Model (추가) ---
    from kfp_workflow.plugins.my_new_model import MyNewModelPlugin
    registry[MyNewModelPlugin.name()] = MyNewModelPlugin

    return registry
```

## Result 데이터클래스 상세

각 스테이지의 출력은 정해진 데이터클래스 형식을 따릅니다:

### LoadDataResult

```python
@dataclass
class LoadDataResult:
    data_dir: str           # 데이터 디렉토리 경로
    dataset_name: str       # 데이터셋 식별자
    num_train_samples: int  # 학습 샘플 수
    num_test_samples: int   # 테스트 샘플 수
    metadata: Dict[str, Any] = field(default_factory=dict)
```

### PreprocessResult

```python
@dataclass
class PreprocessResult:
    artifacts_dir: str      # .npy 파일이 저장된 디렉토리
    x_train_path: str       # 학습 입력 경로
    y_train_path: str       # 학습 레이블 경로
    x_val_path: str         # 검증 입력 경로
    y_val_path: str         # 검증 레이블 경로
    x_test_path: str        # 테스트 입력 경로
    y_test_path: str        # 테스트 레이블 경로
    feature_dim: int        # 입력 특성 차원
    seq_len: int            # 시퀀스 길이
    num_train: int          # 학습 윈도우 수
    num_val: int            # 검증 윈도우 수
    num_test: int           # 테스트 윈도우 수
    metadata: Dict[str, Any] = field(default_factory=dict)
```

### TrainResult

```python
@dataclass
class TrainResult:
    model_path: str         # 저장된 모델 경로 (.pt)
    best_epoch: int         # 최적 에폭
    train_loss: float       # 학습 손실
    val_loss: float         # 검증 손실 (선택 기준 메트릭)
    metadata: Dict[str, Any] = field(default_factory=dict)
```

### EvalResult

```python
@dataclass
class EvalResult:
    metrics: Dict[str, float]  # 평가 메트릭 (rmse, mae 등)
    model_path: str            # 모델 경로 (패스스루)
    metadata: Dict[str, Any] = field(default_factory=dict)
```

### SaveResult

```python
@dataclass
class SaveResult:
    saved_path: str         # 최종 저장 경로
    model_name: str         # 모델 이름
    model_version: str      # 모델 버전
```

## Config 스키마 설계 지침

### extra="allow" 사용

```python
class MyModelConfig(BaseModel):
    known_param: int = 64
    model_config = ConfigDict(extra="allow")  # 알 수 없는 필드 허용
```

`extra="allow"`는 다음 이점을 제공합니다:
- 알려진 필드는 타입 검증
- 알 수 없는 필드는 경고만 (오류 아님)
- 실험적 파라미터를 스키마 변경 없이 전달 가능

### 기본값 제공

모든 필드에 기본값을 제공하여, 최소한의 스펙으로도 동작하도록 합니다:

```python
class MyModelConfig(BaseModel):
    hidden_dim: int = 128    # 기본 128
    num_layers: int = 4      # 기본 4
    dropout: float = 0.1     # 기본 0.1
```
