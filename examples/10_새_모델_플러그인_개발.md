# 10. 새 모델 플러그인 개발

## 목표

루트 `kfp-workflow`에서 새 모델을 사용하려면 루트 플러그인 어댑터를 추가해야 합니다.

## 최소 작업

1. `src/kfp_workflow/plugins/` 아래에 새 플러그인 구현
2. `ModelPlugin` 계약 충족
3. `src/kfp_workflow/plugins/__init__.py`의 레지스트리에 등록
4. 관련 스펙 예제와 테스트 추가
5. 루트 문서 업데이트

## 루트 플러그인이 맡는 일

- 루트 스펙을 모델 패키지 호출 형태로 변환
- `load_data`, `preprocess`, `train`, `evaluate`, `save_model`, `predict` 구현
- 선택적으로 HPO 메서드 구현

## 체크리스트

- 모델 이름 문자열이 일관적인가
- 서빙 시 `predict` 경로가 준비되어 있는가
- `spec validate`에서 필요한 모델별 config 검증이 가능한가
- 튜닝 search space와 objective가 루트 기대치와 맞는가

## 주의

패키지 디렉토리가 존재하는 것만으로는 루트 통합이 되지 않습니다. 현재 `timemixer-new`가 그 예시입니다.
