# 07. 레지스트리 관리

## 목적

루트 프로젝트는 파일 기반 모델 레지스트리와 PVC-backed 데이터셋 레지스트리를 제공합니다.

## 모델 레지스트리

등록:

```bash
kfp-workflow registry model register \
  --name mambasl-cmapss \
  --version v1 \
  --uri /mnt/models/mambasl-cmapss/v1
```

조회:

```bash
kfp-workflow registry model get --name mambasl-cmapss --version v1
kfp-workflow registry model list
```

## 데이터셋 레지스트리

등록:

```bash
kfp-workflow registry dataset register \
  --name cmapss \
  --version v1 \
  --pvc-name dataset-store \
  --subpath cmapss
```

조회:

```bash
kfp-workflow registry dataset get --name cmapss --version v1
kfp-workflow registry dataset list
```

## 언제 쓰는가

- 스펙에서 직접 PVC 경로를 하드코딩하고 싶지 않을 때
- 데이터셋/모델 버전을 명시적으로 관리하고 싶을 때

## 참고

레지스트리 구현은 루트 코드의 `src/kfp_workflow/registry/`에 있습니다. 모델 레지스트리는 JSON 파일을 직접 다루고, 데이터셋 레지스트리는 데이터 PVC 위의 JSON 파일을 사용합니다.
