# 07. 레지스트리 관리

## 개요

kfp-workflow는 파일 기반의 경량 레지스트리 시스템을 제공하여, 학습된 모델과 데이터셋의 메타데이터를 추적합니다. 레지스트리는 PVC 위의 JSON 파일로 저장되므로 외부 데이터베이스가 필요 없으며, 파이프라인 컴포넌트 내부에서도 직접 접근할 수 있습니다.

### 설계 철학

- **PVC 기반**: 모든 메타데이터는 PVC 위의 JSON 파일에 저장
- **외부 의존성 없음**: 별도의 DB 서버나 외부 서비스 불필요
- **파이프라인 통합**: `save_model` 스테이지에서 자동 등록
- **CLI 접근**: 수동 등록, 조회, 목록 조회 가능
- **데이터셋 경로 통합**: `storage.data_subpath`가 비어 있으면, 파이프라인 `load_data` 단계가 같은 data PVC 안의 데이터셋 레지스트리에서 `dataset.name` / `dataset.version` 경로를 조회

## 모델 레지스트리

### 모델 등록 (`registry model register`)

```bash
kfp-workflow registry model register \
    --name mambasl-cmapss \
    --version v1 \
    --uri /mnt/models/mambasl-cmapss/v1/model.pt \
    --framework pytorch \
    --description "MambaSL C-MAPSS FD001 baseline"
```

출력:
```json
{
  "name": "mambasl-cmapss",
  "version": "v1",
  "uri": "/mnt/models/mambasl-cmapss/v1/model.pt",
  "framework": "pytorch",
  "description": "MambaSL C-MAPSS FD001 baseline",
  "registered_at": "2026-03-23T10:00:00"
}
```

**옵션:**

| 옵션 | 필수 | 설명 | 기본값 |
|------|------|------|--------|
| `--name` | O | 모델 이름 | — |
| `--version` | O | 모델 버전 | — |
| `--uri` | O | 모델 아티팩트 경로 (PVC 경로, 플러그인별 파일명) | — |
| `--framework` | X | ML 프레임워크 | `pytorch` |
| `--description` | X | 모델 설명 | `""` |
| `--registry-path` | X | 레지스트리 파일 경로 | `/mnt/models/.model_registry.json` |

### 모델 조회 (`registry model get`)

```bash
# 최신 버전 조회
kfp-workflow registry model get --name mambasl-cmapss

# 특정 버전 조회
kfp-workflow registry model get --name mambasl-cmapss --version v1
```

### 모델 목록 조회 (`registry model list`)

```bash
kfp-workflow registry model list
```

출력 예시:
```
┌────────────────────────────────────────────────────────────────┐
│                            Models                              │
├──────────────────┬─────────┬───────────┬──────────────────────┤
│ NAME             │ VERSION │ FRAMEWORK │ URI                  │
├──────────────────┼─────────┼───────────┼──────────────────────┤
│ mambasl-cmapss   │ v1      │ pytorch   │ /mnt/models/mamba... │
│ mambasl-cmapss   │ v2      │ pytorch   │ /mnt/models/mamba... │
└──────────────────┴─────────┴───────────┴──────────────────────┘
```

JSON 출력:

```bash
kfp-workflow --json registry model list
```

## 데이터셋 레지스트리

### 데이터셋 등록 (`registry dataset register`)

데이터셋 레지스트리는 **같은 data PVC 내부의 하위 경로 선택**에 사용됩니다. 현재 파이프라인 경로는 레지스트리로 다른 PVC를 자동 마운트하지는 않습니다.

```bash
kfp-workflow registry dataset register \
    --name cmapss \
    --pvc-name workspace-data-pvc \
    --subpath cmapss/CMAPSSData \
    --version v1 \
    --description "NASA C-MAPSS dataset root"
```

출력:
```json
{
  "name": "cmapss",
  "version": "v1",
  "pvc_name": "workspace-data-pvc",
  "subpath": "cmapss/CMAPSSData",
  "description": "NASA C-MAPSS dataset root",
  "registered_at": "2026-03-23T10:00:00"
}
```

레지스트리 조회 키는 파이프라인 스펙의 `dataset.name` / `dataset.version`과 정확히 일치해야 합니다. FD001 같은 세부 선택은 `dataset.config.fd[]`에 두고, 레지스트리에는 공통 데이터 루트(`cmapss`)를 등록합니다.

**옵션:**

| 옵션 | 필수 | 설명 | 기본값 |
|------|------|------|--------|
| `--name` | O | 데이터셋 이름 | — |
| `--pvc-name` | O | 데이터가 저장된 PVC 이름 | — |
| `--subpath` | O | PVC 내 데이터 경로 | — |
| `--version` | X | 데이터셋 버전 | `v1` |
| `--description` | X | 데이터셋 설명 | `""` |
| `--registry-path` | X | 레지스트리 파일 경로 | `/mnt/data/.dataset_registry.json` |

### 데이터셋 조회 (`registry dataset get`)

```bash
# 최신 버전
kfp-workflow registry dataset get --name cmapss

# 특정 버전
kfp-workflow registry dataset get --name cmapss --version v1
```

### 데이터셋 목록 조회 (`registry dataset list`)

```bash
kfp-workflow registry dataset list
```

출력 예시:
```
┌──────────────────────────────────────────────────────────────────────┐
│                            Datasets                                  │
├──────────────┬─────────┬──────────────────────┬─────────────────────┤
│ NAME         │ VERSION │ PVC                  │ SUBPATH             │
├──────────────┼─────────┼──────────────────────┼─────────────────────┤
│ cmapss       │ v1      │ workspace-data-pvc   │ cmapss/CMAPSSData   │
│ softs-data   │ v2      │ workspace-data-pvc   │ softs/preprocessed   │
└──────────────┴─────────┴──────────────────────┴─────────────────────┘
```

경로 해석 우선순위는 다음과 같습니다:
1. `storage.data_subpath`가 비어 있지 않으면 그 값을 그대로 사용
2. 비어 있으면 같은 data PVC의 dataset registry에서 `dataset.name` / `dataset.version` 조회
3. 레지스트리에도 없으면 `storage.data_mount_path` 자체를 사용

## 파이프라인 자동 등록

파이프라인의 `save_model` 스테이지에서 모델이 자동으로 레지스트리에 등록됩니다. `MambaSLCmapssPlugin.save_model()` 메서드는 다음을 수행합니다:

1. 학습된 모델 파일(`model.pt`)을 최종 경로로 복사
2. 모델 설정(`model_config.json`)을 함께 복사
3. `FileModelRegistry`에 메타데이터 자동 등록

등록 경로는 스펙의 `storage.model_mount_path`에서 결정됩니다:

```yaml
# 파이프라인 스펙에서
storage:
  model_mount_path: /mnt/models
  # → 레지스트리 파일: /mnt/models/.model_registry.json
```

## 커스텀 레지스트리 경로

클러스터 외부에서 테스트하거나 다른 경로를 사용하려면 `--registry-path` 옵션을 사용합니다:

```bash
# 로컬 파일 기반 테스트
kfp-workflow registry model register \
    --name test-model \
    --version v1 \
    --uri ./models/test/model.pt \
    --registry-path ./test_registry.json

kfp-workflow registry model list \
    --registry-path ./test_registry.json
```

## 문제 해결

### 레지스트리 파일이 없는 경우

첫 번째 `register` 호출 시 자동으로 생성됩니다. 별도의 초기화가 필요 없습니다.

### PVC 접근 불가

레지스트리 경로가 PVC에 있는 경우, 해당 PVC가 마운트된 환경에서만 접근할 수 있습니다:
- 파이프라인 컴포넌트 내부 (자동 마운트)
- PVC를 마운트한 개발용 Pod
