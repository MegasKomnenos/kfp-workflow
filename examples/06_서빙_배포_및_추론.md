# 06. 서빙 배포 및 추론

## 개요

학습이 완료된 모델을 KServe InferenceService로 배포하여 실시간 추론 서비스를 제공할 수 있습니다. kfp-workflow의 `serve` 명령어 그룹은 InferenceService의 생성, 조회, 상태 확인, 삭제를 지원합니다.

### KServe란?

KServe는 Kubernetes 위에서 ML 모델 서빙을 표준화한 플랫폼입니다. 주요 특징:

- **자동 스케일링**: 트래픽에 따라 Pod 수를 자동 조절 (0으로 축소 포함)
- **카나리 배포**: 새 모델 버전을 점진적으로 배포
- **다중 프레임워크**: PyTorch, TensorFlow, ONNX 등 지원
- **표준 API**: REST/gRPC 추론 인터페이스 제공

## 서빙 스펙 파일

서빙 배포는 별도의 YAML 스펙 파일로 정의합니다:

```yaml
# configs/serving/mambasl_cmapss_serve.yaml
metadata:
  name: mambasl-cmapss-fd001

namespace: kubeflow-user-example-com

# 모델 저장소 PVC
model_pvc: workspace-model-pvc
model_subpath: mambasl-cmapss/v1/model.pt

# 서빙 런타임
runtime: kserve-custom       # custom predictor 사용
predictor_image: "your-registry/kfp-workflow:latest"
model_name: mambasl-cmapss

# 스케일링
replicas: 1

# 리소스
resources:
  cpu_request: "2"
  cpu_limit: "4"
  memory_request: 4Gi
  memory_limit: 8Gi
```

### 스펙 필드 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `metadata.name` | O | InferenceService 이름 |
| `namespace` | O | K8s 네임스페이스 |
| `model_pvc` | O | 모델이 저장된 PVC 이름 |
| `model_subpath` | O | PVC 내 모델 파일 경로 |
| `runtime` | O | 서빙 런타임 (예: `kserve-custom`) |
| `predictor_image` | O | 커스텀 프레딕터 Docker 이미지 |
| `model_name` | O | 플러그인 레지스트리에서의 모델 이름 |
| `replicas` | X | 복제본 수 (기본: 1) |
| `resources` | X | CPU/메모리 리소스 설정 |

## 서빙 명령어

### InferenceService 생성 (`serve create`)

```bash
kfp-workflow serve create \
    --spec configs/serving/mambasl_cmapss_serve.yaml
```

출력:
```json
{
  "apiVersion": "serving.kserve.io/v1beta1",
  "kind": "InferenceService",
  "metadata": {
    "name": "mambasl-cmapss-fd001",
    "namespace": "kubeflow-user-example-com"
  },
  ...
}
```

#### Dry Run (미리보기)

실제로 적용하지 않고 생성될 매니페스트만 확인합니다:

```bash
kfp-workflow serve create \
    --spec configs/serving/mambasl_cmapss_serve.yaml \
    --dry-run
```

### InferenceService 목록 조회 (`serve list`)

```bash
kfp-workflow serve list
```

출력 예시:
```
┌──────────────────────────────────────────────────────────────────────┐
│                       InferenceServices                              │
├──────────────────────┬───────┬──────────────────────────┬───────────┤
│ NAME                 │ READY │ URL                      │ CREATED   │
├──────────────────────┼───────┼──────────────────────────┼───────────┤
│ mambasl-cmapss-fd001 │ True  │ http://mambasl-cmapss... │ 2026-03-… │
│ mambasl-cmapss-fd003 │ False │                          │ 2026-03-… │
```

**옵션:**

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--namespace` | K8s 네임스페이스 | `kubeflow-user-example-com` |

```bash

```

출력:
```

```

## 추론 요청

InferenceService가 `Ready` 상태가 되면 REST API로 추론 요청을 보낼 수 있습니다.

### 클러스터 내부에서

```bash
SERVICE_URL="http://mambasl-cmapss-fd001.kubeflow-user-example-com.svc.cluster.local"

curl -X POST "${SERVICE_URL}/v1/models/mambasl-cmapss:predict" \
    -H "Content-Type: application/json" \
    -d '{
        "instances": [
            [[0.1, 0.2, ...], [0.3, 0.4, ...], ...]
        ]
    }'
```

### 클러스터 외부에서 (포트포워딩)

```bash
# InferenceService Pod 포트포워딩
kubectl port-forward -n kubeflow-user-example-com \
    svc/mambasl-cmapss-fd001 8080:80

# 추론 요청
curl -X POST "http://localhost:8080/v1/models/mambasl-cmapss:predict" \
    -H "Content-Type: application/json" \
    -d @input.json
```

### Python 클라이언트 예제

```python
import json
import numpy as np
import requests

# 입력 데이터 준비 (shape: [batch, seq_len, features])
input_data = np.random.randn(1, 50, 14).tolist()

response = requests.post(
    "http://localhost:8080/v1/models/mambasl-cmapss:predict",
    json={"instances": input_data},
)

predictions = response.json()["predictions"]
print(f"예측 RUL: {predictions[0]:.1f} 사이클")
```

## 서빙 상태 모니터링

### Ready 상태 확인

InferenceService가 완전히 준비되려면 세 가지 조건이 모두 `True`여야 합니다:

| 조건 | 설명 |
|------|------|
| `PredictorReady` | 프레딕터 Pod가 준비됨 |
| `IngressReady` | Ingress 경로가 구성됨 |
| `Ready` | 전체 서비스가 준비됨 (위 조건 모두 충족) |

### kubectl로 직접 확인

```bash
# InferenceService 상태
kubectl get inferenceservice -n kubeflow-user-example-com

# Pod 상태 확인
kubectl get pods -n kubeflow-user-example-com -l serving.kserve.io/inferenceservice=mambasl-cmapss-fd001

# Pod 로그 확인
kubectl logs -n kubeflow-user-example-com -l serving.kserve.io/inferenceservice=mambasl-cmapss-fd001 -c kserve-container
```

## 문제 해결

### Ready가 False인 경우

```bash
# 상세 조건 확인
kfp-workflow serve get --name mambasl-cmapss-fd001

# Pod 이벤트 확인
kubectl describe pod -n kubeflow-user-example-com \
    -l serving.kserve.io/inferenceservice=mambasl-cmapss-fd001
```

**일반적인 원인:**
- PVC가 존재하지 않거나 마운트 실패
- Docker 이미지 Pull 실패
- 리소스 부족 (CPU/메모리)
- 모델 파일이 지정된 경로에 없음

### 이미지 Pull 실패

```bash
# 이미지가 레지스트리에 있는지 확인
docker pull your-registry/kfp-workflow:latest

# K8s Secret 확인 (프라이빗 레지스트리)
kubectl get secrets -n kubeflow-user-example-com | grep regcred
```
