# 01. 설치 및 설정

## 사전 요구사항

- Python 3.9 이상
- pip (패키지 관리자)
- Git
- Kubernetes 클러스터 접근 (파이프라인 실행 시 필요)

## 1단계: 저장소 클론 및 디렉토리 이동

```bash
cd /path/to/project
cd test/
```

## 2단계: 가상환경 생성

```bash
make venv
```

이 명령은 `.venv/` 디렉토리에 Python 가상환경을 생성합니다.

직접 생성하려면:
```bash
python3 -m venv .venv
```

## 3단계: 패키지 설치

```bash
make install
```

이 명령은 다음을 수행합니다:
1. pip, setuptools, wheel 업그레이드
2. `kfp-workflow` 패키지와 개발 의존성 설치 (`pip install -e ".[dev]"`)

설치되는 주요 의존성:
- `kfp==2.15.0` — Kubeflow Pipelines SDK
- `kubernetes` — Kubernetes Python 클라이언트
- `typer` + `rich` — CLI 프레임워크
- `pydantic` — 스펙 검증

## 4단계: 가상환경 활성화

```bash
source .venv/bin/activate
```

## 5단계: CLI 설치 확인

```bash
kfp-workflow --help
```

다음과 유사한 출력이 표시되어야 합니다:

```
Usage: kfp-workflow [OPTIONS] COMMAND [ARGS]...

  KFP v2 training pipeline and KServe serving workflow manager.

Options:
  --json    Output in JSON format.
  --help    Show this message and exit.

Commands:
  cluster   Cluster bootstrapping operations.
  pipeline  Compile and submit training pipelines.
  registry  Register and retrieve models and datasets.
  serve     Create and manage KServe InferenceServices.
  spec      Spec validation.
```

## 6단계: 테스트 실행

```bash
make test
```

또는:

```bash
python -m pytest tests/ -v
```

모든 테스트가 통과하면 설치가 정상적으로 완료된 것입니다.

## 7단계: Kubernetes 환경 확인 (선택사항)

파이프라인 제출 및 서빙 배포를 위해 Kubernetes 클러스터가 필요합니다:

```bash
# 클러스터 연결 확인
kubectl cluster-info

# Kubeflow 네임스페이스 확인
kubectl get pods -n kubeflow

# KFP API 서비스 확인
kubectl get svc -n kubeflow | grep ml-pipeline
```

## 시스템 전역 CLI 설정 (선택사항)

가상환경 활성화 없이 `kfp-workflow` 명령어를 사용하려면 심볼릭 링크를 생성합니다:

```bash
sudo ln -sf $(pwd)/.venv/bin/kfp-workflow /usr/local/bin/kfp-workflow
```

## 문제 해결

### `kfp-workflow: command not found`
가상환경이 활성화되었는지 확인하세요:
```bash
source .venv/bin/activate
which kfp-workflow
```

### 의존성 충돌
가상환경을 재생성하세요:
```bash
rm -rf .venv
make venv
make install
```
