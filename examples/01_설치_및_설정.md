# 01. 설치 및 설정

## 목표

루트 프로젝트를 로컬에서 설치하고 기본 검증까지 끝내는 것이 목표입니다.

## 요구사항

- Python `>=3.10`
- `python3 -m venv` 사용 가능 환경
- Docker
- `kubectl`
- Kubeflow/KServe 클러스터 접근이 필요한 경우 올바른 kubeconfig

## 로컬 설치

```bash
make venv
make install
```

## 기본 검증

```bash
make test
make spec-validate
make compile-pipeline
```

또는 직접 실행:

```bash
kfp-workflow spec validate --spec configs/pipelines/sample_train.yaml

kfp-workflow pipeline compile \
  --spec configs/pipelines/sample_train.yaml \
  --output pipelines/sample_train.yaml
```

## 자주 발생하는 문제

### `kfp-workflow` 명령을 찾지 못함

가상환경이 활성화되지 않았거나 editable install이 끝나지 않은 경우입니다.

```bash
. .venv/bin/activate
make install
```

### 테스트가 의존성 문제로 실패함

루트 프로젝트는 선택형 extra 없이 `.[dev]` 설치만으로 기본 테스트를 실행하도록 구성되어 있습니다. 가상환경을 새로 만들고 다시 설치하는 편이 가장 빠릅니다.
