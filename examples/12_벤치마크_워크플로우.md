# 12. 벤치마크 워크플로우

## 벤치마크 모델

루트 벤치마크는 세 요소를 묶습니다.

- `model`: 임시 KServe `InferenceService`
- `scenario`: 입력 생성과 재생 방식
- `metrics`: 실행 중 수집 결과

## 기본 명령

```bash
kfp-workflow benchmark compile \
  --spec configs/benchmarks/mambasl_cmapss_kepler_smoke.yaml \
  --output pipelines/mambasl_cmapss_kepler_smoke.yaml

kfp-workflow benchmark submit \
  --spec configs/benchmarks/mambasl_cmapss_kepler_smoke.yaml
```

결과 조회:

```bash
kfp-workflow benchmark list
kfp-workflow benchmark get <run-id-or-prefix>
kfp-workflow benchmark download <run-id-or-prefix>
```

## 포함된 예제

### `mambasl_cmapss_kepler_smoke.yaml`

- 임시 MambaSL 서빙 생성
- 일부 FD001 데이터를 순차 재생
- Kepler energy 메트릭 수집

### `mambasl_cmapss_test.yaml`

- 임시 MambaSL 서빙 생성
- FD001 test set 평가
- `f1_score`, `precision`, `recall`, `accuracy` 계산

## 재사용 가능한 구성

`configs/benchmarks/` 아래에는 다음 재사용 단위가 있습니다.

- `scenarios/`
- `metrics/`

벤치마크 spec은 이 참조 파일이나 Python 정의를 조합해 구성될 수 있습니다.

## 준비 절차

```bash
kfp-workflow cluster bootstrap \
  --type benchmark \
  --spec configs/benchmarks/mambasl_cmapss_kepler_smoke.yaml
```

필요 시 벤치마크 실행 중 사용될 모델이 PVC에 이미 존재해야 합니다.

## 참고

벤치마크 결과는 결과 PVC에 저장되며, `benchmark download`가 이를 로컬 파일로 내려받습니다.
- 예제 스펙은 `cleanup: true`를 사용하지만, 임시 서빙 정리 여부는 `model.cleanup` 값으로 결정됩니다.
